## 正文（中文）

```
工作 8 年，我的最大杀手锏是下属周报里的「问号」。

但每周 1 小时全花在追问下属上，太累。
所以我用 Claude Code 给自己造了个秘书 —— Workpad。

【它干什么】
- 你给它一份周报 + 一段自我介绍
- 它输出一份本地 HTML：
  · 3 视角表（OKR / 专项 / 人）
  · 每行事项配 AI 分析 + 评论草稿
  · 评论草稿可以直接贴飞书

【为什么不是 ChatGPT 一问就行】
试了 3 个月，纯对话 LLM 写的评论有 3 个问题：
1. 飘 —— 不问具体数字
2. 偷懒 —— 5 行事项 owner 都 @ 同一个人
3. 不给问号 —— 自己预设了答案

Workpad 的核心是 6 条「死规矩」（D1-D6），是我自己 callout 8 个版本沉淀出来的纪律。

【免费 + 开源】
- GitHub：github.com/xllinbupt/workpad（仓库地址换成你的）
- Claude Code 一键装：./install.sh
- 数据全在本地，下属名 / OKR 不会离开你电脑

【适合谁】
- 带团队的产品 / 运营 / 技术 leader
- 已经在用 Claude Code 的人（没装的可以装一下，免费）
- 不愿意把公司数据贴给在线 AI 的人

评论区留下你的痛点，我看后续 v2 加什么功能 ❤️

#Claude #AI工作流 #开源工具 #管理 #周报 #效率工具
```

## 评论钩子（自问自答置顶）

> Q：我没用过 Claude Code 啊，能用吗？
> A：能。先去 claude.com/claude-code 装个免费版，然后 clone 我这个仓库跑 install.sh。前后不到 5 分钟。

> Q：会上传我的周报吗？
> A：不会。Workpad 是个 Skill（本地配置）+ Claude 客户端 + 你自己的 API。整个链路里你的周报只走过你 Claude 那一次推理。

> Q：评论真的能直接贴飞书吗？
> A：评论草稿是文本，你复制粘贴到飞书评论框就行。下个版本会做 OAuth 一键同步。
